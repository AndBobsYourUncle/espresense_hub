# Deployment

Scripts for running ESPresense Hub as a systemd service on a Debian/Ubuntu
host (bare metal, VM, or LXC container).

## Layout

| Path | Purpose |
|---|---|
| (wherever you cloned) | Repo + build output. Owned by `espresense` so updates can run without `sudo` for git/npm. |
| `/var/lib/espresense-hub/config.yaml` | Your config — edit this, never `config.yaml` in the repo. |
| `/var/lib/espresense-hub/calibration.json` | Per-pair stats, GT samples, residuals. Auto-written every 60 s. |
| `/var/lib/espresense-hub/devices.json` | Per-device pins + bias accumulators. Auto-written. |
| `/etc/systemd/system/espresense-hub.service` | Generated by `install.sh` based on your install location. |

The repo and the runtime state live in different directories on purpose —
`git pull` in the repo never touches your config or accumulated calibration.

## First-time install

Clone the repo wherever you want it to live (commonly `/opt`, but it works
anywhere — your home dir, a different mount, whatever):

```bash
sudo apt-get install -y git
sudo git clone https://github.com/your-user/espresense_hub.git /opt/espresense-hub
cd /opt/espresense-hub
sudo ./deploy/install.sh
```

The script:
1. Installs Node.js 20 from NodeSource (skip with `SKIP_NODE=1` if already present).
2. Creates an unprivileged `espresense` system user.
3. Creates `/var/lib/espresense-hub` and seeds `config.yaml` from
   `config.example.yaml` (only on first install — never overwrites).
4. Sets the repo dir owner to `espresense`.
5. Runs `npm ci && npm run build`.
6. Generates and installs the systemd unit pointing at the actual repo
   location and your chosen settings.
7. Enables (but doesn't start, on first install) the service so you can
   edit config first.

After install:

```bash
sudo nano /var/lib/espresense-hub/config.yaml   # at minimum: mqtt.host, floors, nodes
sudo systemctl start espresense-hub
sudo journalctl -u espresense-hub -f            # watch logs
```

The hub is reachable at `http://<host>:3000`. Put nginx/Caddy/Traefik in
front if you want TLS or a different external port.

### Customizing the install

Override defaults via env vars to `install.sh`:

```bash
sudo PORT=8080 BIND_ADDR=127.0.0.1 STATE_DIR=/srv/espresense ./deploy/install.sh
```

Available knobs: `STATE_DIR`, `SERVICE_USER`, `SERVICE_NAME`, `PORT`,
`BIND_ADDR`, `SKIP_NODE`.

## Updates

To pull the latest from `main` and rebuild:

```bash
sudo /opt/espresense-hub/deploy/update.sh
```

The script:
1. Fetches `origin`.
2. Bails if there's nothing new.
3. `git reset --hard origin/main` (your config and state are untouched —
   they're in `/var/lib/espresense-hub`, not the repo).
4. Re-runs `npm ci && npm run build`.
5. Restarts the service and prints status.

Different branch:

```bash
sudo BRANCH=develop /opt/espresense-hub/deploy/update.sh
```

If you change settings later (port, bind address, state dir, service name),
re-run `install.sh` with the new env vars — it regenerates the unit file.

## Service management

```bash
sudo systemctl status espresense-hub        # is it running?
sudo systemctl restart espresense-hub       # apply config edits
sudo systemctl stop espresense-hub
sudo journalctl -u espresense-hub -f        # live logs
sudo journalctl -u espresense-hub --since "1 hour ago"
```

## Uninstall

```bash
sudo systemctl stop espresense-hub
sudo systemctl disable espresense-hub
sudo rm /etc/systemd/system/espresense-hub.service
sudo rm /etc/sudoers.d/espresense-hub
sudo systemctl daemon-reload
sudo rm -rf /opt/espresense-hub                  # or wherever you cloned
# Keep state, or nuke it:
# sudo rm -rf /var/lib/espresense-hub
sudo userdel espresense
```

## Notes

- **Backups**: snapshot `/var/lib/espresense-hub/` to preserve calibration —
  the accumulated per-pair stats and per-device pin biases take days/weeks
  to rebuild. The repo at the install location is reproducible from
  `git clone`.
- **Config path**: the generated unit sets `ESPRESENSE_CONFIG_PATH`
  pointing the app at `/var/lib/espresense-hub/config.yaml`. The state
  files (`calibration.json`, `devices.json`) are always written next to
  that path.
